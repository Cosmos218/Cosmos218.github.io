Kd-Trees

  • Key is 2d(point in a plane).
  • Space-partitioning trees: use in games.

2d tree construction

  • Data Structure: BST, but key is x and y coordinate
  • When construction, first divide by horizontal, then vertical, then horizontal, then vertical…. it’s like a binary search in 2d space!
  • At this time, we shall realize that BST’s searching is just kind of binary search, but its constrution is more flexible and smart than array and linked list!

Nearest neighbor search in a 2d tree demo

Search through a 2d tree to find the nearest neighbor, when the current min record is less than the distance between query and rect of the node, stop searching that branch.

Kd-Trees assignment

Specification
Some tricky points:

  1. When writing nearest how to implement without using global variables? That is, how to do recursive and pass mutable Double arguments?
  2. Do not compute the distance between the query point and the point in a node if the closest point discovered so far is closer than the distance between the query point and the rectangle corresponding to the node.

Method implemention algorithm

  • size(): simply do size++ when insert()
  • insert(): use helper method like put() in BST, recursively find the best place to insert Node; to compute rectangle, need to pass parent node when recurse
  • contains(): similar to get() in BST
  • range(): use helper function to recursively search. Several different conditions: rect contains x.p, rect cross the line of x, rect on the lb of line, rect on the rt of the line.
  • nearest(): have to deal with the problem that no need to compute when current min is less than query to rect of the node

Main gain in knowledge: be familiar with tree searching and adding recursive operation, have a try of the 2d geometry operation
